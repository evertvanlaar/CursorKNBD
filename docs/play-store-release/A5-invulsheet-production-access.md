## Q2 — How did you recruit? (296 tekens)

*Enige veld met volledige decline/2nd-test context.*

```
Testers joined our closed test via the Play Console opt-in link. I also shared APK builds with family and others for on-device feedback. After our first production decline, this was our 2nd 14-day closed test. As the developer I ran on-device checks for stability, UI/layout and listing accuracy.
```

---

## Q3 — Engagement

```
Testers used core flows: directory, business pages, favorites, bus times, Pelion Guide. Not every tester utilized every feature, but all main areas are covered. Usage matched real visitors (short task sessions). Some explored all screens; typical users pick 1-2 features. EN/EL.
```

---

## Q4 — Feedback + how collected (266 tekens)

```
Stable, no crashes; directory, favorites, bus times and Pelion Guide valued. Minor issues: install help, weather widget, Android 15 layout. Collected via closed-test feedback, APK testers, and my release notes. Fixed across multiple releases; re-tested on release 7.
```

## Q5 — Intended audience (219 tekens)

```
Tourists and residents in Kala Nera, Kato Gatzea, Koropi and Pelion visitors (Greece). People finding places to eat, stay and shop, local services, and trip info (bus times, flights, events, numbers). English and Greek.
```

## Q6 — Value to users

```
Helps tourists and residents discover local places to eat, stay and shop and local contact details; save favorites on device; check estimated Kala Nera bus times (not official KTEL); use Pelion Guide (flights, events, numbers). Play TWA for kalanera.gr. Internet required; favorites local. EN/EL.
```
